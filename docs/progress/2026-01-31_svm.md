## 2. Feature Extraction from IMU Signals

### 2.1 Input Data

The raw IMU signals are segmented into fixed-length **sliding windows**.  
Each window contains **6 signal channels**, corresponding to accelerometer and gyroscope axes.

### 2.2 Extracted Features

For each window, statistical features are extracted from every channel:

- Mean value
- Standard deviation
- Maximum value
- Minimum value
- Signal energy (mean squared value)

As a result:
- **5 features per channel**
- **6 channels**
- **Total: 30 features per window**

After feature extraction:
- Total number of samples: **56,233**
- Feature vector dimension: **30**

### 5.1 Analysis

- **Low thresholds (âˆ’1.0, âˆ’0.5):**
  - Very high recall for Fall detection
  - Low precision, resulting in many false alarms
  - Overall accuracy significantly decreases

- **High thresholds (0.5, 1.0):**
  - High precision
  - Recall drops sharply, leading to many missed fall events

- No threshold provides a good balance between high fall recall and acceptable overall accuracy.

---

## 6. Overall Evaluation of the SVM Model

The Linear SVM model demonstrates the following characteristics:

- Suitable as a **baseline model**
- Limited ability to model nonlinear decision boundaries
- High trade-off between recall and precision when tuning the decision threshold

As fall detection is a safety-critical application, missing fall events is unacceptable.  
Therefore, **the Linear SVM model is not selected as the final model for the fall detection system**, but it provides valuable insights for comparative analysis.
